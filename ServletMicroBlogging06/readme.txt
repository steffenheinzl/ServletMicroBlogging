Neu hinzugekommen:
Servlet Login
login.html 

Es gibt eine neue Datei login.html und ein Servlet namens Login, das den Wunsch zum Einloggen entgegennimmt und bei Erfolg den Benutzernamen in einer Session speichert.
Die Login-Form übermittelt lediglich einen Benutzernamen. Auf eine Passwortabfrage wird in diesem Beispiel verzichtet.
Der Benutzer soll nach erfolgreichem Login zurück an die index.html redirected werden.
